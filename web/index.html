Why QuantaDB?

Built for performance, designed for simplicity, engineered for the future.

Lightning Fast

Built with Rust for maximum performance and memory safety. In-memory operations with optional file-based persistence.

QuantaSQL

Custom SQL dialect designed for simplicity and clarity. Essential operations without the complexity of traditional SQL.

Multiple Clients

CLI client, Python connector, Rust library, and desktop application. Choose the interface that fits your workflow.

Memory Safe

Rust's ownership model eliminates entire classes of bugs. No segfaults, no data races, no memory leaks.

Persistent Storage

File-based persistence with JSON serialization. Your data survives server restarts and system reboots.

Scalable Architecture

Modular design with async networking. Ready for clustering, replication, and distributed deployments.

Current State

What's working today in QuantaDB v0.1.0

✅ Implemented

  • Custom SQL Parser (QuantaSQL)
  • In-Memory Storage Engine
  • File-Based Persistence
  • TCP Server with Async I/O
  • CLI Client (psql-like)
  • Python Connector (PyO3)
  • Rust Client Library
  • Desktop Client (Tauri)
  • Basic CRUD Operations
  • WHERE Clause Filtering
  • Type System with Validation
  • Graceful Shutdown

Roadmap

The future of QuantaDB - where we're heading

v0.2.0 - Enhanced Features

  • UPDATE statements
  • JOIN operations
  • Aggregate functions (COUNT, SUM, AVG)
  • Indexes for faster queries
  • Transaction support

v0.3.0 - Advanced Storage

  • B+ Tree storage engine
